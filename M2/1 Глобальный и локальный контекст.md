### 0. Введение: почему это важно

**Академическое определение:**  
*Область видимости* (scope) — это часть программы, в которой идентификатор (переменная, функция, класс) остаётся доступным и имеет своё значение.

**Сленговый перевод:**  
«Область видимости — это как комната в общежитии: внутри ты можешь делать что хочешь, но переменные из соседней комнаты не видны, пока не попросишь ключ».

Для понимания декораторов, замыканий и многих продвинутых паттернов необходимо чётко понимать правила видимости и способы их изменения.

---

### 1. Иерархия областей видимости (LEGB Rule)

Python использует **LEGB-правило** для поиска переменных (по возрастанию приоритета):

| Уровень | Расшифровка | Что содержит |
|---------|-------------|---------------|
| **L**ocal | Локальная | Переменные, определённые внутри функции |
| **E**nclosing | Охватывающая | Переменные из внешних (вложенных) функций |
| **G**lobal | Глобальная | Переменные, определённые на уровне модуля |
| **B**uilt-in | Встроенная | Встроенные имена (print, len, open и т.д.) |

**Золотое правило:** при обращении к переменной Python сначала ищет её в локальной области, затем в охватывающих (по цепочке), затем в глобальной, и наконец — во встроенной.

**Сленг:** «LEGB — как лестница: сначала смотришь под ногами (local), потом на этаж выше (enclosing), потом на улицу (global), а в последнюю очередь — в справочник (built-in)».

### 2. Локальные vs глобальные переменные

#### 2.1 Базовое поведение: чтение глобальной переменной

```python
name = "Илья"  # глобальная переменная (уровень модуля)

def say_bye():
    # Ищет name в локальной области → не находит
    # Поднимается на уровень выше (global) → находит
    print("Пока,", name)

def say_hi():
    name = "Иван"  # создаёт НОВУЮ локальную переменную
    print("Привет,", name)

say_hi()   # Привет, Иван
say_bye()  # Пока, Илья

print(name)  # Илья — глобальная не изменилась
```

**Что здесь происходит:**
- В `say_hi()` переменная `name` создаётся **локально** и "затеняет" (shadows) глобальную.
- В `say_bye()` локальной переменной `name` нет, поэтому Python берёт глобальную.
- Глобальная переменная `name` остаётся неизменной.

**Важное следствие:** внутри функции можно *читать* глобальные переменные, но *присваивание* по умолчанию создаёт новую локальную.

---

#### 2.2 Изменение глобальной переменной: ключевое слово `global`

Если мы хотим *изменить* глобальную переменную внутри функции, нужно явно объявить её как `global`:

```python
name = "Илья"

def say_hi():
    global name           # 🌍 говорим: "мы работаем с глобальной переменной"
    name = "Иван"         # теперь изменяем глобальную
    print("Привет,", name)

def say_bye():
    print("Пока,", name)

say_hi()   # Привет, Иван
say_bye()  # Пока, Иван — имя изменилось глобально!

print(name)  # Иван — глобальная действительно изменена
```

**Академическая ремарка:**  
`global` — это директива, которая говорит интерпретатору, что последующие присваивания имени `name` должны относиться к переменной на уровне модуля, а не создавать локальную.

**Сленг:** «Global — как вездеходный пропуск: ты можешь менять правила игры во всей программе. Но с великой силой приходит великая ответственность — не злоупотребляй».

### 3. Вложенные функции и `nonlocal`

#### 3.1 Проблема: затенение переменной внешней функции

Когда у вас есть функция внутри функции (вложенная), возникает третий уровень видимости — **enclosing** (охватывающий):

```python
x = 0                     # global

def outer():
    x = 1                 # enclosing (для inner)
    
    def inner():
        x = 2             # local (для inner)
        print("inner:", x)
    
    inner()
    print("outer:", x)

outer()
print("global:", x)

# inner: 2
# outer: 1
# global: 0
```

**Анализ:**  
- `inner()` создаёт свою локальную `x = 2`
- `outer` не знает об этом (у неё своя `x = 1`)
- Глобальная `x = 0` остаётся нетронутой

---

#### 3.2 Решение: `nonlocal` для доступа к enclosing scope

Если внутри `inner()` мы хотим изменять переменную `x` из `outer()` (не глобальную, а именно из охватывающей функции), используем `nonlocal`:

```python
x = 0

def outer():
    x = 1                 # переменная из enclosing scope
    
    def inner():
        nonlocal x        # 🔗 привязываемся к x из outer
        x = 2             # изменяем x в outer
        print("inner:", x)
    
    inner()
    print("outer:", x)

outer()
print("global:", x)

# inner: 2
# outer: 2   ← изменилось!
# global: 0  ← не изменилось
```

**Что произошло:**
- `nonlocal x` в `inner()` говорит: "не создавай новую локальную переменную, а используй ту, что находится в ближайшем enclosing scope (функция `outer`)"
- Присваивание `x = 2` изменяет переменную `outer`
- Глобальная `x = 0` остаётся без изменений

---

#### 3.3 nonlocal не работает с глобальными переменными

```python
x = 0

def outer():
    # Пытаемся привязаться к глобальной через nonlocal
    nonlocal x   # ❌ SyntaxError: no binding for nonlocal 'x' found
    x = 1

# Ошибка! На уровне выше (global) — это уже не enclosing scope
```

**Правило:** `nonlocal` работает только для переменных из enclosing scope (вложенных функций), но **не для глобальных**. Для глобальных используйте `global`.

---

#### 3.4 Цепочка вложенных функций: nonlocal поднимается по ступенькам

```python
def outer():
    value = "original"
    
    def middle():
        value = "middle_local"
        
        def inner():
            nonlocal value   # к какой переменной привяжется?
            value = "changed"
            print("inner:", value)
        
        inner()
        print("middle:", value)
    
    middle()
    print("outer:", value)

outer()
```

**Результат:**
```
inner: changed
middle: changed   ← изменилась переменная из middle
outer: original   ← outer не изменилась
```

**Объяснение:**  
`nonlocal` привязывается к **ближайшей** enclosing переменной с таким именем (поднимается по иерархии, но не до глобального уровня). В данном случае это `value` из `middle()`.

Если бы в `middle()` не было своей `value`, `nonlocal` поднялся бы до `outer()`.

### 4. Сравнение: global vs nonlocal

| Характеристика | `global` | `nonlocal` |
|----------------|----------|------------|
| **Применяется к** | переменным на уровне модуля | переменным в enclosing scope (вложенные функции) |
| **Где используется** | в любой функции | только во вложенных функциях |
| **Ошибка если переменной нет** | создаёт новую глобальную | `SyntaxError` |
| **Поднимается по цепочке** | сразу на верхний (модуль) | на ближайший enclosing scope |
| **Типичное использование** | конфиги, счётчики вызовов | замыкания, декораторы, фабрики функций |

### 5. Реальные кейсы (из практики senior-разработчика)

#### 5.1 Счётчик вызовов функции (замыкание)

```python
def create_counter():
    count = 0                     # переменная в enclosing scope
    
    def counter():
        nonlocal count            # изменяем её
        count += 1
        return count
    
    return counter

# Создаём два независимых счётчика
calls1 = create_counter()
calls2 = create_counter()

print(calls1())  # 1
print(calls1())  # 2
print(calls2())  # 1 (независимый!)
print(calls1())  # 3
```

**Где применяется:**  
- Декораторы для подсчёта вызовов
- Кэширование (memoization)
- Генерация уникальных ID

**Сленг:** «Замыкание с nonlocal — это как карман в куртке: каждый экземпляр функции получает свой собственный карман, и никто чужой туда не залезет».

---

#### 5.2 Простейший декоратор с сохранением состояния

```python
def log_calls(func):
    call_count = 0
    
    def wrapper(*args, **kwargs):
        nonlocal call_count
        call_count += 1
        print(f"Функция {func.__name__} вызвана {call_count} раз(а)")
        return func(*args, **kwargs)
    
    return wrapper

@log_calls
def say_hello(name):
    print(f"Hello, {name}!")

say_hello("Alice")  # Функция say_hello вызвана 1 раз(а) | Hello, Alice!
say_hello("Bob")    # Функция say_hello вызвана 2 раз(а) | Hello, Bob!
```

#### 5.3 Антипаттерн: чрезмерное использование global

```python
# ❌ ПЛОХО: глобальные переменные как "чёрная магия"
status = "stopped"

def start():
    global status
    status = "running"

def stop():
    global status
    status = "stopped"

# Где-то в другом файле, через 500 строк кода...
start()
# Кто изменил status? Как он теперь называется?
```

**Почему плохо:**
- Сложно отлаживать (кто и когда меняет глобальную переменную?)
- Нарушает инкапсуляцию
- Мешает многопоточности (race conditions)

**✅ ЛУЧШЕ:** передавать состояние явно или использовать классы.

```python
class Service:
    def __init__(self):
        self.status = "stopped"
    
    def start(self):
        self.status = "running"
    
    def stop(self):
        self.status = "stopped"
```

**Золотое правило senior'а:**  
Если вам кажется, что `global` — это хорошее решение, проверьте, можно ли обойтись без него. В 95% случаев — можно.

### 6. Частые ошибки и ловушки

#### ❌ Ошибка 1: присваивание глобальной переменной без `global`

```python
count = 0

def increment():
    count += 1   # ❌ UnboundLocalError: local variable 'count' referenced before assignment

increment()
```

**Почему ошибка:** Python видит присваивание `count += 1` и считает `count` локальной переменной. Но при этом пытается прочитать её значение до присваивания.

**Исправление:** Добавить `global count`.

---

#### ❌ Ошибка 2: nonlocal там, где нет enclosing переменной

```python
def func():
    nonlocal x   # ❌ SyntaxError: no binding for nonlocal 'x' found
    x = 10
```

**Исправление:** либо создать enclosing scope (вложенную функцию), либо использовать `global`.

---

#### ❌ Ошибка 3: путаница между чтением и записью

```python
data = [1, 2, 3]

def modify():
    # 🟢 Меняем содержимое объекта — global не нужен
    data.append(4)
    
    # 🔴 Присваиваем новое значение переменной — нужен global
    data = [5, 6, 7]  # создаёт локальную переменную!

modify()
print(data)  # [1, 2, 3, 4] — добавление сработало, но присваивание нет!
```

**Правило:**  
- Мутация объекта (вызов методов) — не требует `global`
- Переназначение переменной (`=`) — требует `global` или `nonlocal`

### 7. Визуализация LEGB (для запоминания)

```python
# BUILT-IN (встроенные: print, len, open...)
#     ↑
# GLOBAL (уровень модуля)
#     ↑
# ENCLOSING (охватывающие функции)
#     ↑
# LOCAL (локальная область)
```

**Проверочный пример:**

```python
from math import pi   # импорт в глобальную область

def outer():
    pi = 3.14          # enclosing
    
    def inner():
        pi = 3.14159   # local
        print(pi)      # находит local
    
    inner()
    print(pi)          # находит enclosing (3.14)

outer()
print(pi)              # находит global (math.pi ≈ 3.141592653589793)
```

**Результат:**
```
3.14159
3.14
3.141592653589793
```

### 8. Сводная таблица: когда что использовать

| Ситуация | Решение | Пример |
|----------|---------|--------|
| Прочитать глобальную переменную | ничего (просто читаем) | `print(name)` |
| Изменить глобальную переменную | `global` | `global x; x = 10` |
| Прочитать переменную из enclosing | ничего (просто читаем) | `print(x)` (где x из outer) |
| Изменить переменную из enclosing | `nonlocal` | `nonlocal x; x = 10` |
| Прочитать встроенную функцию | ничего (доступна всегда) | `len([1,2,3])` |

### Итоговая мысль

> Области видимости — это не просто теория из учебника. Это база, на которой строятся замыкания, декораторы, фабрики функций и многие другие паттерны. Пока вы не поймёте `global` и `nonlocal`, вы будете писать баги, связанные с неожиданным изменением переменных. А как только поймёте — станете писать код, который другие разработчики будут цитировать как пример чистоты и ясности.

**Сленговое резюме:**
«Global — как крик на всю программу: "Эй, все, слушайте сюда!" Nonlocal — как шёпот на ухо родительской функции. А LEGB — это карта метро, по которой Python ищет твои переменные. Выучи маршрут — и никогда не заблудишься в коде».

```python
# Финальная демонстрация: три уровня видимости в одном примере
level = "🌍 global"

def outer():
    level = "📦 enclosing"
    
    def inner():
        level = "🔧 local"
        print(f"In inner: {level}")
    
    inner()
    print(f"In outer: {level}")

outer()
print(f"Global: {level}")

# In inner: 🔧 local
# In outer: 📦 enclosing
# Global: 🌍 global
```
