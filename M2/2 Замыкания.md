### 0. Введение: динамическая генерация функций

**Академическое определение:**
В функциональном программировании стандартной практикой является динамическая генерация функциональных объектов в процессе исполнения кода с их последующим вызовом. Этот подход позволяет создавать гибкие, параметризуемые и переиспользуемые абстракции.

**Ключевые концепции модуля:**

- Замыкание (closure)
- Частичное применение (partial application)
- Карринг (currying)
- Декоратор (decorator)
- Функтор (functor)

В этом занятии мы сфокусируемся на фундаменте всех этих концепций — **замыканиях**.

**Сленговый перевод:**
«Замыкание — это функция, которая помнит, где родилась, даже когда её унесли далеко от дома».

---

### 1. Функции внутри функций: первый шаг к замыканию

В Python функции являются объектами первого класса, поэтому их можно определять внутри других функций.

#### 1.1 Простейший пример: вызов внутренней функции

```python
def say_name(name):
    def say_goodbye():
        print("Don't say me goodbye, " + name + "!")
  
    say_goodbye()  # вызываем внутреннюю функцию

say_name("Sergey")  # Don't say me goodbye, Sergey!
```

**Что здесь происходит:**

1. Вызывается внешняя функция `say_name("Sergey")`
2. Создаётся внутренняя функция `say_goodbye()`
3. Внутренняя функция немедленно вызывается
4. После завершения `say_goodbye` управление возвращается в `say_name`
5. `say_name` завершается

**Ничего необычного** — стек вызовов работает предсказуемо.

---

#### 1.2 Возврат функции вместо вызова

А теперь — магия:

```python
def say_name(name):
    def say_goodbye():
        print("Don't say me goodbye, " + name + "!")
  
    return say_goodbye  # ⬅️ возвращаем функцию, а не вызываем

# Сохраняем возвращённую функцию
f = say_name("Sergey")

# Теперь f — это функция say_goodbye, которая "помнит" name="Sergey"
f()  # Don't say me goodbye, Sergey!
```

**Ключевое наблюдение:**
Когда мы вызываем `say_name("Sergey")`, внешняя функция выполняется и *завершается*. Локальная переменная `name` должна была бы исчезнуть. Но `say_goodbye` **продолжает иметь к ней доступ**!

**Академическая ремарка:**
Это возможно потому, что Python создаёт *замыкание* — объект, содержащий ссылку на функцию и её *лексическое окружение* (переменные из внешней области видимости).

**Сленг:**
«Функция-призрак: тело умерло, но память осталась».

### 2. Как работают замыкания: внутренний механизм

#### 2.1 Что такое замыкание (closure)

**Определение (строгое):**
*Замыкание* — это функция, которая запоминает окружение, в котором она была создана, даже после того, как это окружение формально завершило своё существование.

**Технически замыкание состоит из:**

1. **Функции** (код, который будет выполнен)
2. **Ссылок на переменные** из внешнего окружения (non-local variables)

```python
def outer(x):
    def inner(y):
        return x + y
    return inner

closure = outer(10)
print(closure(5))  # 15
```

**Как это выглядит в памяти:**

```
Замыкание (closure)
├── Код функции inner
└── Лексическое окружение (cell objects)
    └── x = 10  (захваченная переменная)
```

---

#### 2.2 Почему переменные не исчезают: механизм ссылок

```python
def say_name(name):
    def say_goodbye():
        print("Don't say me goodbye, " + name + "!")
    return say_goodbye

f = say_name("Sergey")  # f хранит ссылку на внутреннюю функцию
```

**Что происходит под капотом:**

1. Создаётся объект функции `say_goodbye` внутри `say_name`
2. Python видит, что `say_goodbye` использует переменную `name` из внешней области
3. Переменная `name` помещается в специальную структуру — **ячейку замыкания** (cell object)
4. Объект функции `say_goodbye` получает ссылку на эту ячейку
5. Даже после завершения `say_name`, ячейка продолжает существовать, потому что на неё есть ссылка из `say_goodbye`
6. Когда мы сохраняем `f = say_name("Sergey")`, создаётся ещё одна ссылка на `say_goodbye`
7. Сборщик мусора не удаляет ячейку, пока существует хотя бы одна ссылка на функцию

**Следствие:** Каждый вызов внешней функции создаёт *новое* замыкание с *собственным* набором захваченных переменных:

```python
f1 = say_name("Sergey")
f2 = say_name("Python")

f1()  # Don't say me goodbye, Sergey!
f2()  # Don't say me goodbye, Python!
# Полностью независимые замыкания!
```

### 3. Замыкания с изменяемым состоянием: `nonlocal`

#### 3.1 Проблема: изменение захваченной переменной

```python
def counter(start=0):
    def step():
        start += 1  # ❌ UnboundLocalError
        return start
    return step

c = counter(10)
c()  # Ошибка!
```

**Почему ошибка?**
Внутри `step()` есть присваивание `start += 1`. Python по умолчанию считает `start` локальной переменной, если она где-то присваивается. Но она не определена локально — отсюда ошибка.

**Сленг:**
«Python путается: то ли ты хочешь использовать внешнюю переменную, то ли создать свою собственную».

---

#### 3.2 Решение: `nonlocal`

```python
def counter(start=0):
    def step():
        nonlocal start  # 🔗 "Я буду менять внешнюю переменную!"
        start += 1
        return start
    return step

c1 = counter(10)
c2 = counter()  # start=0 по умолчанию

print(c1(), c2())  # 11 1
print(c1(), c2())  # 12 2
print(c1(), c2())  # 13 3
```

**Важные моменты:**

- `nonlocal` работает только для переменных из ближайшего enclosing scope
- Нельзя использовать `nonlocal` для глобальных переменных (нужен `global`)
- `nonlocal` создаёт привязку не к значению, а к *ячейке памяти* переменной

**Академическая ремарка:**
`nonlocal` — это explicit declaration, которая говорит интерпретатору: "ищи эту переменную не в локальной области, а в охватывающих (enclosing) областях". Без неё присваивание создало бы новую локальную переменную, а не изменило бы захваченную.

### 4. Реальные примеры использования замыканий

#### 4.1 Фабрика функций-счётчиков

```python
def create_counter(initial=0, step=1):
    """Создаёт счётчик с заданным начальным значением и шагом"""
    count = initial
  
    def increment():
        nonlocal count
        count += step
        return count
  
    def decrement():
        nonlocal count
        count -= step
        return count
  
    def reset():
        nonlocal count
        count = initial
        return count
  
    def get():
        return count
  
    # Возвращаем словарь методов (мини-класс!)
    return {
        'inc': increment,
        'dec': decrement,
        'reset': reset,
        'get': get
    }

# Использование
counter = create_counter(10, 2)
print(counter['inc']())  # 12
print(counter['inc']())  # 14
print(counter['dec']())  # 12
print(counter['get']())  # 12
counter['reset']()
print(counter['get']())  # 10
```

**Сленг:**
«Замыкание — это бедный родственник класса. Класс — богатый дядя с методами и наследованием. Оба делают одно и то же, просто по-разному».

---

#### 4.2 Функция для удаления символов (strip с предустановкой)

```python
def strip_string(strip_chars=" "):
    """Создаёт функцию для удаления указанных символов"""
    def do_strip(string):
        return string.strip(strip_chars)
    return do_strip

# Создаём разные "ножи" для обрезки строк
strip_default = strip_string()                    # удаляем только пробелы
strip_punctuation = strip_string(" !?,.;:")       # удаляем знаки препинания
strip_digits = strip_string("0123456789")         # удаляем цифры

# Применяем
test_string = "  hello world!!!...  "

print(strip_default(test_string))      # "hello world!!!..."
print(strip_punctuation(test_string))  # "  hello world  "
print(strip_digits("123abc456"))       # "abc"
```

**Почему это круто:**

- Не нужно каждый раз передавать `strip_chars`
- Код становится декларативным: `strip_punctuation` читается как намерение
- Функции можно передавать в `map`, `filter` без лишних параметров

#### 4.3 Ленивая загрузка ресурсов (caching)

```python
def lazy_loader(resource_getter):
    """Загружает ресурс только при первом обращении"""
    resource = None
  
    def loader():
        nonlocal resource
        if resource is None:
            print("Loading resource...")  # имитация дорогой операции
            resource = resource_getter()
        return resource
  
    return loader

# Дорогой ресурс (big dataset, database connection, etc.)
def get_expensive_data():
    print("Actually loading 1GB of data...")
    return [1, 2, 3, 4, 5]

data_loader = lazy_loader(get_expensive_data)

# Первый вызов — загружаем
print(data_loader())  # Loading resource... / Actually loading... / [1,2,3,4,5]

# Второй вызов — берём из кэша
print(data_loader())  # [1,2,3,4,5] (без загрузки)
```

#### 4.4 Кэширование результатов (memoization)

```python
def memoize(func):
    """Простейший кэш для результатов функции"""
    cache = {}
  
    def wrapper(*args):
        if args not in cache:
            print(f"Вычисляем {func.__name__}{args}...")
            cache[args] = func(*args)
        else:
            print(f"Берём из кэша {func.__name__}{args}...")
        return cache[args]
  
    return wrapper

@memoize  # это декоратор, построенный на замыкании!
def fibonacci(n):
    if n <= 1:
        return n
    return fibonacci(n-1) + fibonacci(n-2)

print(fibonacci(5))
# Вычисляем fibonacci(5)...
# Вычисляем fibonacci(4)...
# ... много вычислений ...
# 5

print(fibonacci(5))
# Берём из кэша fibonacci(5)... ← моментально!
```

### 5. Замыкания и циклы: классическая ловушка

#### 5.1 Проблема: позднее связывание (late binding)

```python
functions = []
for i in range(3):
    def func():
        return i
    functions.append(func)

for f in functions:
    print(f())  # 2 2 2  ← ожидали 0, 1, 2?
```

**Почему?**
Переменная `i` не захватывается *по значению*, а захватывается *по ссылке*. Все три функции используют одну и ту же переменную `i`, которая после цикла равна `2`.

**Сленг:**
«Все функции смотрят на одну и ту же миску с едой — что в миске в момент вызова, то и получают».

---

#### 5.2 Решение 1: параметр по умолчанию

```python
functions = []
for i in range(3):
    def func(x=i):  # x захватывает ТЕКУЩЕЕ значение i
        return x
    functions.append(func)

for f in functions:
    print(f())  # 0, 1, 2
```

**Как работает:**
Параметры по умолчанию вычисляются в момент определения функции, а не в момент вызова.

---

#### 5.3 Решение 2: дополнительное замыкание

```python
functions = []
for i in range(3):
    def make_func(x):
        def func():
            return x
        return func
    functions.append(make_func(i))

for f in functions:
    print(f())  # 0, 1, 2
```

### 6. Замыкания vs Классы: когда что использовать

| Аспект                                  | Замыкание                              | Класс                                            |
| --------------------------------------------- | ----------------------------------------------- | ----------------------------------------------------- |
| **Синтаксис**                  | Короткий, функциональный  | Многословный, объектный          |
| **Состояние**                  | Одна-две переменные            | Много полей                                 |
| **Методы**                        | Простые функции                   | Полноценные методы                   |
| **Наследование**            | Нет                                          | Есть                                              |
| **Интроспекция**            | Сложная (`.__closure__`)               | Простая (`.` attribute)                      |
| **Память**                        | Минимальный оверхед           | Больше на класс                          |
| **Когда использовать** | Простые счётчики, фильтры | Сложные объекты с поведением |

**Пример сравнения:**

```python
# Замыкание
def make_multiplier(factor):
    return lambda x: x * factor

doubler = make_multiplier(2)

# Класс
class Multiplier:
    def __init__(self, factor):
        self.factor = factor
    def __call__(self, x):
        return x * self.factor

doubler_class = Multiplier(2)
```

**Сленг:**
«Замыкание — это класс с одним методом `__call__`, написанный на минималках. Замыкания для простоты, классы для архитектуры».

### 7. Интроспекция замыканий: что внутри?

```python
def outer(x, y):
    def inner(z):
        return x + y + z
    return inner

closure = outer(10, 20)

# Проверяем, что это замыкание
print(closure.__closure__)      # (<cell at 0x...: int object at 0x...>, <cell at 0x...: int object at 0x...>)
print(closure.__code__.co_freevars)  # ('x', 'y')
print(closure.__code__.co_varnames)  # ('z',)

# Получаем значения захваченных переменных
cell_x, cell_y = closure.__closure__
print(cell_x.cell_contents)  # 10
print(cell_y.cell_contents)  # 20
```

**Сленг:**
«`__closure__` и `__code__.co_freevars` — это рентген замыкания. Можешь заглянуть внутрь и посмотреть, что там спрятано».

### 8. Сводная таблица: ключевые концепции

| Концепция                                           | Описание                                                                                                  | Пример                                                 |
| ------------------------------------------------------------ | ----------------------------------------------------------------------------------------------------------------- | ------------------------------------------------------------ |
| **Замыкание**                                 | Функция + захваченное окружение                                                        | `def outer(x): def inner(y): return x+y; return inner`     |
| **nonlocal**                                           | Разрешает изменять захваченные переменные                                   | `nonlocal x; x += 1`                                       |
| **Лексическое окружение**          | Переменные из внешней области видимости                                        | переменная `name` в примере `say_name` |
| **Ячейка (cell)**                                | Контейнер для захваченной переменной                                             | `cell.cell_contents`                                       |
| **Свободная переменная (free var)** | Переменная, используемая в замыкании, но определённая вне его | `x` в `inner`                                           |

### Итоговая мысль

> Замыкания — это мост между процедурным и функциональным программированием. Они позволяют инкапсулировать состояние без создания полноценных классов, писать элегантные декораторы и строить мощные абстракции. Как только вы освоите замыкания, мир функционального программирования в Python откроется для вас во всей красе.

**Сленговое резюме:**
«Замыкание — это функция с секретами. Создал, настроил, забыл о внутренностях. А она помнит и продолжает работать, даже когда всё вокруг уже умерло. Идеальный инструмент для ленивых архитекторов и умных декораторов».

```python
# Финальная демонстрация: цепочка замыканий
def trace_closure(name, value):
    print(f"Создаётся замыкание '{name}' со значением {value}")
  
    def closure():
        print(f"Вызвано замыкание '{name}', хранящее значение {value}")
        return value
  
    return closure

a = trace_closure("A", 42)
b = trace_closure("B", 100)

a()  # Вызвано замыкание 'A', хранящее значение 42
b()  # Вызвано замыкание 'B', хранящее значение 100
```
