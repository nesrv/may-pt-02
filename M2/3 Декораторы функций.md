### 0. Введение: что такое декорирование

**Академическое определение:**
*Декорирование* — это механизм, позволяющий обернуть один вызываемый объект (функцию или класс) другим вызываемым объектом для расширения или модификации его поведения без изменения исходного кода. Декораторы реализуют паттерн проектирования "Декоратор" (Decorator pattern) на синтаксическом уровне языка.

**Сленговый перевод:**
«Декоратор — это такой фантик для функции: оборачиваешь её в конфетную бумажку, а внутри всё та же вкусная функция».

**Ключевые идеи:**

1. Декоратор — это функция, которая принимает функцию и возвращает функцию
2. Декораторы используют механизм замыканий для доступа к оригинальной функции
3. Синтаксис `@decorator` — это синтаксический сахар для `func = decorator(func)`

---

### 1. Анатомия простейшего декоратора

#### 1.1 Базовый декоратор без параметров

```python
def func_decorator(func):
    """Простейший декоратор — обёртка без параметров"""
    def wrapper():
        print("--- перед вызовом функции ---")
        func()
        print("--- после вызова функции ---")
  
    return wrapper

# Применяем декоратор
def some_func():
    print("Вызов функции some_func")

# Старый способ (явное оборачивание)
some_func = func_decorator(some_func)
some_func()
# --- перед вызовом функции ---
# Вызов функции some_func
# --- после вызова функции ---
```

**Что здесь происходит:**

```
some_func (оригинал) → func_decorator() → wrapper() (новая функция)
                                              ↓
                                        вызывает оригинал
```

**Академическая ремарка:**
Декоратор `func_decorator` — это функция высшего порядка. Она принимает вызываемый объект `func` и возвращает новый вызываемый объект `wrapper`. Переменная `some_func` после присваивания начинает ссылаться на `wrapper`, а оригинальная функция остаётся внутри замыкания.

---

#### 1.2 Синтаксический сахар: @

```python
def func_decorator(func):
    def wrapper():
        print("--- перед вызовом функции ---")
        func()
        print("--- после вызова функции ---")
    return wrapper

@func_decorator      # 🔥 магия! это эквивалентно some_func = func_decorator(some_func)
def some_func():
    print("Вызов функции some_func")

some_func()
# Вывод тот же, что и выше
```

**Важно:** Строка `@func_decorator` должна находиться **непосредственно перед** определением функции. В этот момент (на этапе определения) Python автоматически применяет декоратор.

**Сленг:**
«Собачка `@` — это как печать "одобрено": поставил — и функция завернулась в декоратор сама, без лишних телодвижений».

### 2. Универсальный декоратор: работа с любыми параметрами

#### 2.1 Проблема: декоратор не знает о параметрах функции

```python
def func_decorator(func):
    def wrapper():           # нет параметров
        print("--- перед вызовом ---")
        func()
        print("--- после вызова ---")
    return wrapper

@func_decorator
def greet(name):             # функция с параметром!
    print(f"Hello, {name}!")

greet("Alice")   # ❌ TypeError: wrapper() takes 0 positional arguments but 1 was given
```

**Причина:** `wrapper()` не принимает аргументы, а `greet` теперь ссылается на `wrapper`.

---

#### 2.2 Решение: `*args` и `**kwargs`

```python
def func_decorator(func):
    def wrapper(*args, **kwargs):           # принимает любые аргументы
        print("--- перед вызовом функции ---")
        result = func(*args, **kwargs)      # передаёт их оригиналу
        print("--- после вызова функции ---")
        return result                       # возвращает результат
  
    return wrapper

# Теперь работает с любыми функциями
@func_decorator
def greet(name):
    print(f"Hello, {name}!")

@func_decorator
def add(a, b):
    return a + b

@func_decorator
def print_all(*args, **kwargs):
    print(f"args: {args}, kwargs: {kwargs}")

greet("Alice")                               # ✅ работает
result = add(10, 20)                         # ✅ 30
print_all(1, 2, 3, x=10, y=20)              # ✅ работает
```

**Анатомия универсального декоратора:**

```python
def decorator(func):
    def wrapper(*args, **kwargs):
        # Действия ДО вызова оригинальной функции
        result = func(*args, **kwargs)
        # Действия ПОСЛЕ вызова оригинальной функции
        return result
    return wrapper
```

**Сленг:**
«`*args, **kwargs` — это как карманный мультитул для декоратора: суёшь туда что угодно, на выходе получаешь то же самое, но обёрнутое в логику».

### 3. Декораторы с доступом к возвращаемому значению

Если функция что-то возвращает, декоратор должен это передать дальше:

```python
def logger(func):
    def wrapper(*args, **kwargs):
        print(f"📞 Вызов функции '{func.__name__}'")
        result = func(*args, **kwargs)      # сохраняем результат
        print(f"✅ Функция '{func.__name__}' вернула: {result}")
        return result                       # возвращаем результат
  
    return wrapper

@logger
def multiply(a, b):
    return a * b

@logger
def divide(a, b):
    if b == 0:
        return float('inf')
    return a / b

print(multiply(5, 3))   # 📞 → ✅ → 15
print(divide(10, 2))    # 📞 → ✅ → 5.0
```

**Ключевой момент:** Без `return result` декоратор "съел" бы возвращаемое значение, и вызывающий код получил бы `None`.

### 4. Декоратор с проверкой/модификацией аргументов

#### 4.1 Валидация параметров

```python
def validate_age(func):
    def wrapper(*args, **kwargs):
        # Извлекаем параметры (хеуровнево, но для примера пойдёт)
        if len(args) >= 2:
            name, age = args[0], args[1]
        else:
            name = kwargs.get('name', 'Unknown')
            age = kwargs.get('age', 0)
      
        # Валидация
        if age < 0:
            print(f"⚠️ Отрицательный возраст ({age}) заменён на 1")
            age = 1
        elif age > 150:
            print(f"⚠️ Подозрительный возраст ({age}), установлен 100")
            age = 100
      
        # Вызов оригинальной функции с (возможно) изменёнными параметрами
        return func(name, age)
  
    return wrapper

@validate_age
def print_person(name, age):
    print(f"👤 Имя: {name}, Возраст: {age}")

print_person("Tom", 38)      # 👤 Имя: Tom, Возраст: 38
print_person("Bob", -5)      # ⚠️ → 👤 Имя: Bob, Возраст: 1
print_person("Eve", 200)     # ⚠️ → 👤 Имя: Eve, Возраст: 100
```

---

#### 4.2 Модификация возвращаемого значения

```python
def ensure_non_negative(func):
    """Гарантирует, что функция не вернёт отрицательное значение"""
    def wrapper(*args, **kwargs):
        result = func(*args, **kwargs)
        if result < 0:
            print(f"⚠️ Результат {result} заменён на 0")
            return 0
        return result
  
    return wrapper

@ensure_non_negative
def subtract(a, b):
    return a - b

print(subtract(10, 5))   # 5
print(subtract(5, 10))   # ⚠️ → 0
```

### 5. Декоратор-таймер для измерения производительности

Классический пример из реальной практики:

```python
import time
from functools import wraps

def timer(func):
    """Измеряет время выполнения декорируемой функции"""
    @wraps(func)  # сохраняет метаданные функции (имя, docstring)
    def wrapper(*args, **kwargs):
        start = time.perf_counter()        # максимальная точность
        result = func(*args, **kwargs)
        end = time.perf_counter()
        elapsed = end - start
        print(f"⏱️ Функция '{func.__name__}' выполнена за {elapsed:.6f} сек")
        return result
  
    return wrapper

@timer
def fibonacci_recursive(n):
    """Рекурсивное вычисление числа Фибоначчи (медленно)"""
    if n <= 1:
        return n
    return fibonacci_recursive(n-1) + fibonacci_recursive(n-2)

@timer
def fibonacci_iterative(n):
    """Итеративное вычисление числа Фибоначчи (быстро)"""
    a, b = 0, 1
    for _ in range(n):
        a, b = b, a + b
    return a

# Тестируем
print(fibonacci_recursive(35))   # время ~ 2-5 секунд
print(fibonacci_iterative(35))   # время ~ 0.00001 секунд
```

**⚠️ Важное предупреждение (из опыта):**
Декорирование рекурсивных функций может привести к многократному применению декоратора на каждом уровне рекурсии:

```python
@timer
def factorial(n):
    return 1 if n <= 1 else n * factorial(n-1)  # ❌ factorial(n-1) — это уже обёрнутая функция!

factorial(5)
# Будет выведено 5 сообщений от таймера (для n=5,4,3,2,1)
```

**Решение:** Оборачивать только внешний вызов, либо использовать `functools.lru_cache`.

### 6. Декоратор для контроля доступа (авторизация)

```python
# Конфигурация уровня доступа
current_user = "admin"  # может быть "admin", "guest", "user"

def requires_admin(func):
    """Декоратор: разрешает выполнение только администратору"""
    def wrapper(*args, **kwargs):
        if current_user != "admin":
            print(f"⛔ Доступ запрещён! Требуются права администратора.")
            print(f"   Текущий пользователь: {current_user}")
            return None
        print(f"🔓 Доступ разрешён для администратора")
        return func(*args, **kwargs)
  
    return wrapper

@requires_admin
def delete_user(user_id):
    print(f"🗑️ Пользователь {user_id} удалён из базы данных")
    return True

@requires_admin
def change_settings(param, value):
    print(f"⚙️ Настройка {param} изменена на {value}")
    return True

# Проверяем
current_user = "admin"
delete_user(42)        # 🔓 → 🗑️ Пользователь 42 удалён

current_user = "guest"
delete_user(42)        # ⛔ Доступ запрещён!
```

### 7. Несколько декораторов: порядок применения

Декораторы можно **накладывать друг на друга**. Важно понимать порядок:

```python
def bold(func):
    def wrapper():
        return f"<b>{func()}</b>"
    return wrapper

def italic(func):
    def wrapper():
        return f"<i>{func()}</i>"
    return wrapper

@bold
@italic
def hello():
    return "Hello"

# Аналогично: hello = bold(italic(hello))
print(hello())   # <b><i>Hello</i></b>

# Меняем порядок:
@italic
@bold
def hello2():
    return "Hello"

print(hello2())  # <i><b>Hello</b></i>
```

**Правило (важно!):**
Декораторы применяются **снизу вверх** (от ближайшего к функции к дальнему), но выполняются **сверху вниз** при вызове.

```python
@outer
@middle
@inner
def func(): pass

# Эквивалентно:
func = outer(middle(inner(func)))
```

**Сленг:**
«Декораторы — как луковая шелуха: последний надетый (сверху) — самый внешний. Но оборачивать начинали с самого внутреннего».

### 8. Декоратор с аргументами (параметризованный)

Иногда декоратору нужно передавать свои собственные параметры:

```python
def repeat(times):
    """Декоратор: повторяет вызов функции указанное количество раз"""
    def decorator(func):
        def wrapper(*args, **kwargs):
            results = []
            for i in range(times):
                print(f"Попытка {i+1}/{times}")
                results.append(func(*args, **kwargs))
            return results
        return wrapper
    return decorator

@repeat(times=3)   # ← передаём аргумент декоратору!
def greet(name):
    print(f"Hello, {name}!")
    return f"Greeted {name}"

result = greet("Alice")
print(f"Результаты: {result}")
# Попытка 1/3 → Hello, Alice!
# Попытка 2/3 → Hello, Alice!
# Попытка 3/3 → Hello, Alice!
# Результаты: ['Greeted Alice', 'Greeted Alice', 'Greeted Alice']
```

**Анатомия параметризованного декоратора:** (три уровня вложенности!)

```python
def parametrized_decorator(param):
    # 1 уровень: принимает параметры декоратора
    def decorator(func):
        # 2 уровень: принимает декорируемую функцию
        def wrapper(*args, **kwargs):
            # 3 уровень: принимает аргументы вызова
            # ... логика с использованием param ...
            return func(*args, **kwargs)
        return wrapper
    return decorator

# Использование:
@parametrized_decorator(param="value")
def my_func(): pass
```

**Сленг:**
«Параметризованный декоратор — это матрёшка: функция в функции в функции. Три уровня — и на каждом свои задачи».

### 9. Сохранение метаданных: `functools.wraps`

**Проблема:** Декоратор "затирает" имя и документацию функции:

```python
def my_decorator(func):
    def wrapper(*args, **kwargs):
        """Документация wrapper"""
        return func(*args, **kwargs)
    return wrapper

@my_decorator
def greet(name):
    """Приветствует пользователя"""
    print(f"Hello, {name}!")

print(greet.__name__)   # 'wrapper' ← а не 'greet'!
print(greet.__doc__)    # 'Документация wrapper' ← потеряли оригинал!
```

**Решение:** `@wraps` из `functools`:

```python
from functools import wraps

def my_decorator(func):
    @wraps(func)          # 🔥 копирует __name__, __doc__, __module__, __annotations__
    def wrapper(*args, **kwargs):
        """Документация wrapper (уже не важна)"""
        return func(*args, **kwargs)
    return wrapper

@my_decorator
def greet(name):
    """Приветствует пользователя"""
    print(f"Hello, {name}!")

print(greet.__name__)   # 'greet' ← спасено!
print(greet.__doc__)    # 'Приветствует пользователя' ← спасено!
```

**Золотое правило senior'а:**
Всегда используйте `@wraps(func)` внутри любого декоратора, если нет веской причины не делать этого. Иначе отладка превратится в кошмар.

### 10. Класс-декоратор (через `__call__`)

Декоратором может быть не только функция, но и класс:

```python
class TimerDecorator:
    def __init__(self, func):
        self.func = func
        print(f"🔧 Декоратор инициализирован для функции '{func.__name__}'")
  
    def __call__(self, *args, **kwargs):
        """Вызывается вместо оригинальной функции"""
        import time
        start = time.perf_counter()
        result = self.func(*args, **kwargs)
        elapsed = time.perf_counter() - start
        print(f"⏱️ {self.func.__name__}: {elapsed:.6f} сек")
        return result

@TimerDecorator
def slow_calculation(n):
    total = 0
    for i in range(n):
        total += i ** 2
    return total

print(slow_calculation(10_000_000))
# 🔧 Декоратор инициализирован для функции 'slow_calculation'
# ⏱️ slow_calculation: 0.523456 сек
# 333333283333335000000
```

**Когда использовать класс-декоратор:**

- Нужно хранить дополнительное состояние между вызовами
- Нужны вспомогательные методы
- Хочется чётко разделить инициализацию (`__init__`) и вызов (`__call__`)

### 11. Декораторы классов

Декораторы могут применяться и к классам:

```python
def add_repr(cls):
    """Добавляет методу __repr__ удобное представление"""
    def __repr__(self):
        attrs = ', '.join(f"{k}={v}" for k, v in self.__dict__.items())
        return f"{cls.__name__}({attrs})"
  
    cls.__repr__ = __repr__
    return cls

@add_repr
class Person:
    def __init__(self, name, age):
        self.name = name
        self.age = age

p = Person("Alice", 30)
print(p)   # Person(name=Alice, age=30) — без декоратора было бы <__main__.Person object at 0x...>
```

### 12. Частые ошибки и антипаттерны

#### ❌ Забыли `return result`

```python
def bad_decorator(func):
    def wrapper(*args, **kwargs):
        func(*args, **kwargs)   # ❌ нет return
    return wrapper

@bad_decorator
def add(a, b):
    return a + b

print(add(2, 3))   # None
```

#### ❌ Неправильный порядок параметров в декораторе с аргументами

```python
# ❌ Ошибка: забыли один уровень вложенности
def repeat(times, func):   # ← неверная сигнатура
    def wrapper(*args, **kwargs):
        ...
    return wrapper

# ✅ Правильно: три уровня
def repeat(times):
    def decorator(func):
        def wrapper(*args, **kwargs):
            ...
        return wrapper
    return decorator
```

#### ❌ Декорирование рекурсивных функций (внутренние вызовы теряют декоратор)

```python
@timer
def fib(n):
    return n if n <= 1 else fib(n-1) + fib(n-2)  # fib — это уже обёрнутая версия → рекурсия с декоратором

fib(30)  # каждая рекурсия будет замерять время отдельно!
```

**Решение:** Вынести рекурсивную логику в отдельную недекорированную функцию.

### 13. Сводная таблица: виды декораторов

| Тип декоратора               | Синтаксис | Вложенность                        | Применение                                        |
| ----------------------------------------- | ------------------ | --------------------------------------------- | ----------------------------------------------------------- |
| **Без параметров**     | `@deco`          | 2 уровня (deco → wrapper)              | Логирование, тайминг                      |
| **С параметрами**       | `@deco(arg)`     | 3 уровня (deco → decorator → wrapper) | repeat, retry, rate_limit                                   |
| **Класс-декоратор**   | `@DecoClass`     | 1 класс (`__init__`, `__call__`)     | Состояние между вызовами              |
| **Декоратор класса** | `@deco_class`    | 1 функция на класс              | Добавление методов, регистрация |

### Итоговая мысль

> Декораторы — это элегантный способ разделения сквозной функциональности (cross-cutting concerns): логирование, тайминг, кэширование, контроль доступа, валидация. Они позволяют следовать принципу DRY (Don't Repeat Yourself) и сохранять код чистым, не засоряя бизнес-логику вспомогательными операциями.

**Сленговое резюме:**
«Декоратор — как фреймворк для ленивых: написал один раз, навесил собачек на нужные функции, и весь вспомогательный код применился сам. Тайминг, логи, проверки — всё под контролем, а бизнес-логика сияет чистотой».

```python
# Финальная демонстрация: цепочка полезных декораторов
from functools import wraps
import time

def log_call(func):
    @wraps(func)
    def wrapper(*args, **kwargs):
        print(f"📞 Вызвана {func.__name__}")
        return func(*args, **kwargs)
    return wrapper

def timer(func):
    @wraps(func)
    def wrapper(*args, **kwargs):
        start = time.perf_counter()
        result = func(*args, **kwargs)
        print(f"⏱️ {func.__name__}: {time.perf_counter() - start:.6f} сек")
        return result
    return wrapper

@log_call
@timer
def hard_work(n):
    return sum(i**2 for i in range(n))

hard_work(1_000_000)
# 📞 Вызвана hard_work
# ⏱️ hard_work: 0.123456 сек
```
