## Именованные кортежи (`namedtuple`)

**Формально:** `collections.namedtuple` — фабрика подклассов `tuple` с **именованными полями** и фиксированной длиной. Это удобная форма **записи** (record type): иммутабельность кортежа + доступ по имени вместо «магических индексов» `[1]`, которые через неделю никто не помнит.

**По-живому:** лёгкий DTO без бойлерплейта `class` до эпохи датаклассов; в новом коде часто предпочитают `@dataclass(frozen=True)`, но `namedtuple` всё ещё живёт в стандартной библиотеке, тестах и легаси.

---

### Базовый пример

Имя типа (первый аргумент) традиционно задают в **CamelCase** на латинице — так меньше сюрпризов с отладкой и сериализацией.

```python
from collections import namedtuple

Citizen = namedtuple("Citizen", ["name", "age", "status"])
ivan = Citizen(name="Иван Петров", age=27, status="машинист")
print(ivan)
print(ivan.status)
```

Точечная нотация сознательно напоминает доступ к полям объекта — это фича читаемости, а не «полу-класс» с методами (хотя у `namedtuple` есть и служебные методы вроде `_replace`).

---

### Пример 1 — точка в 3D

```python
from collections import namedtuple

Point = namedtuple("Point", ["x", "y", "z"])
p = Point(4, 7, 10)
print("x:", p.x)
print("y:", p.y)
print("z:", p.z)
```

---

### Пример 2 — вложенные записи

```python
from collections import namedtuple

Engine = namedtuple("Engine", ["type", "cylinders"])
Car = namedtuple("Car", ["make", "model", "year", "engine"])

engine_instance = Engine(type="1.5L", cylinders=4)
car_instance = Car(make="Honda", model="City", year=2020, engine=engine_instance)

print("Car Make:", car_instance.make)
print("Car Model:", car_instance.model)
print("Car Year:", car_instance.year)
print("Car Engine Type:", car_instance.engine.type)
print("Car Engine Cylinders:", car_instance.engine.cylinders)
```

### Задача (практика)

Реализуйте **три функции** для работы с текстом (слова удобно нормализовать: нижний регистр, выкинуть «пустые» токены после `split`):

1. Печатает текст и выводит **количество слов**.
2. Возвращает или печатает **словарь** «слово → число вхождений».
3. Возвращает **итератор / список** структур `namedtuple` (например, `TokenCount(word, count)`), сохраняющий **порядок первого появления** слова в тексте.

Пример «грязного» текста для отладки (повторы, лишние пробелы, разный регистр):

```text
Design, develop, maintain and test cloud applications in Python, and document API for cloud services.
design, develop,          and test cloud applications in Python, and document API for       services.
Design,        ,          and      cloud              in Python, and document     for       services.
```

**Подсказка по инженерии:** не копите огромные промежуточные списки без нужды; для подсчёта частот обычно достаточно одного прохода с `Counter`, а порядок первого вхождения можно восстановить вторым проходом по токенам.
