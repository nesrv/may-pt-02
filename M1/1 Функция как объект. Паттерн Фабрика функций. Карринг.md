### 1. Функция как объект первого класса

В парадигме функционального программирования (и в Python, который поддерживает её элементы) функция — это **полноправный объект**. В терминах теории языков программирования это означает, что функция является *объектом первого класса* (first-class citizen):

- её можно присвоить переменной;
- передать как аргумент в другую функцию;
- вернуть как результат;
- хранить в структурах данных (list, dict, tuple).

Иными словами, функция в Python не просто "блок инструкций", а значение со своим типом, идентичностью и возможностью быть переиспользованной по ссылке.

#### Пример: присваивание функции переменной

```python
def say_hello() -> None:
    print("Hello")

def say_goodbye() -> None:
    print("Good Bye")

# Присваиваем функцию, а НЕ результат её вызова
message = say_hello
message()       # Hello

# Переприсваиваем
message = say_goodbye
message()       # Good Bye
```

Обратите внимание: `message = say_hello` — без скобок. Скобки вызовут функцию *сразу*. Такой трюк с передачей ссылки называют **"функцией без вызова"** (func without parens). В народе — "закинули функцию в переменную, как снэк в карман".

Аналогично с параметрами и возвращаемым значением:

```python
def sum(a: int, b: int) -> int:
    return a + b

def multiply(a: int, b: int) -> int:
    return a * b

operation = sum
print(operation(5, 6))      # 11

operation = multiply
print(operation(5, 6))      # 30
```

> **Сленг**: «я скопипастил не результат, а саму функцию — и погнал».
> **Академически**: ссылка на вызываемый объект (callable) сохраняет поведенческую сущность.

### 2. Функция как параметр (higher-order function)

Функции, принимающие другие функции в качестве аргументов, называются **функциями высшего порядка** (higher-order functions). Это фундамент таких механик, как `map`, `filter`, `sorted(key=...)`, декораторы и callback-обработчики.

```python
def do(a: int, b: int, op: callable) -> int:
    """op — бинарная операция (callable с двумя аргументами)"""
    return op(a, b)

def sum(a: int, b: int) -> int:
    return a + b

def multiply(a: int, b: int) -> int:
    return a * b

print(do(5, 5, sum))       # 10
print(do(5, 5, multiply))  # 25
```

Здесь `op` — это *абстракция операции*. Во время проектирования `do` мы не знаем, что именно будет подставлено в рантайме. Это называется **стратегией** (strategy pattern) на функциональных спичках.

> **Сленг**: «прокинули колбэк без нейминга — чистая джедайская магия».
> **Академически**: позднее связывание (late binding) вызываемой сущности.

### 3. Функция как результат (function factory)

Функция также может вернуть другую функцию. Такая техника позволяет создавать **замыкания** (closures) и **фабрики поведения**.

#### Пример 1: простая фабрика

```python
def sum(a: int, b: int) -> int:
    return a + b

def subtract(a: int, b: int) -> int:
    return a - b

def multiply(a: int, b: int) -> int:
    return a * b

def select_operation(choice: int) -> callable:
    if choice == 1:
        return sum
    elif choice == 2:
        return subtract
    else:
        return multiply

operation = select_operation(1)   # operation = sum
print(operation(10, 6))           # 16

operation = select_operation(2)   # subtract
print(operation(10, 6))           # 4
```

> Полезно, когда поведение выбирается динамически (AI-агент, парсер команд, плагин).

#### Пример 2: словарь с функциями (dispatch table)

«Дисклеймер хардкорщика»: `if-elif` цепочки часто заменяются словарём — это быстрее и читаемее для большого числа вариантов.

```python
def func1(num: int) -> int:
    return num + 1

def func2(num: int) -> int:
    return num * 2

my_dict = {'func1': func1, 'func2': func2}

print(my_dict['func1'](2))   # 3
print(my_dict['func2'](2))   # 4
```

#### Пример 3: паттерн «Фабрика функций» с защитой от отсутствия ключа

```python
calc = {
    "plus": lambda x, y: x + y,
    "minus": lambda x, y: x - y,
    "division": lambda x, y: x / y,
    "power": pow               # встроенная функция
}

def action(match: str, dictionary: dict, default: str = "NO CALC") -> callable:
    """Возвращает функцию из словаря или fallback, возвращающий default"""
    if match in dictionary:
        return dictionary[match]
    return lambda *x: default   # универсальный заглушка

plus = action('plus', calc)
minus = action('minus', calc)
power = action('power', calc)
square = action('square', calc)

print(plus(5, 4))        # 9
print(minus(5, 4))       # 1
print(power(3, 3))       # 27
print(square(1, 1))      # 'NO CALC'
print(square(1))         # 'NO CALC' — за счёт *x
```

**Где реально применяется:**

- парсеры форматов (json / yaml / toml / csv);
- обработчики команд в телеграм/чат-ботах;
- кодогенерация и плагинные архитектуры.

«Старый инженер подтвердит»: фабрика функций убивает copy-paste и позволяет не душить код условиями на 500 строк.

### 4. Карринг (currying)

Карринг — это преобразование функции с несколькими аргументами в цепочку вложенных функций, каждая из которых принимает **один** аргумент.
Название — в честь Хаскелла Карри (хотя идея старше). В Python карринг не встроен в синтаксис, но легко эмулируется через замыкания.

#### Классический пример

```python
def greet_curried(greeting: str):
    def greet(name: str) -> None:
        print(f"{greeting}, {name}")
    return greet

greet_hello = greet_curried("Hello")   # фиксируем приветствие
greet_hello("Игорь")                   # Hello, Игорь
greet_hello("Роман")                   # Hello, Роман

# Прямой вызов без промежуточной переменной (чистый карринг)
greet_curried("Hi")("Сергей")          # Hi, Сергей
```

#### Академический комментарий

Карринг даёт частичное применение (partial application) без изменения исходной функции.В Python для частичного применения чаще используют `functools.partial`, но карринг — более фундаментальное понятие из λ-исчисления.

> **Сленг**: «закаррировал приветствие, получил шаблонный проперти».
> **Примечание сеньора**: карринг реально удобен при композиции пайпов и в data pipelines.
