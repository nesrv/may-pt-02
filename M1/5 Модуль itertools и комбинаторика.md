### 0. Введение: зачем программисту комбинаторика

**Академическое определение:**
*Комбинаторика* — раздел дискретной математики, изучающий конечные множества и их подмножества, комбинаторные конфигурации (сочетания, перестановки, размещения) и способы их подсчёта.

**Сленговый перевод:**
«Комбинаторика — это когда ты перебираешь все возможные варианты, потому что "А что, если попробовать вот так?"».

В реальной разработке комбинаторика встречается везде:

- Генерация тестовых данных (все возможные комбинации параметров)
- Проверка гипотез в data science
- Поиск оптимальных решений (брутфорс)
- Криптография и теория кодирования
- UI-комбинаторика (все состояния интерфейса)

Python даёт два мощных инструмента:

1. `math` — для подсчёта *количества* комбинаций
2. `itertools` — для *генерации* комбинаций

### 1. Базовые понятия: выборы в мире пиццы

Представьте, что у вас есть 4 начинки для пиццы:

```python
from math import comb, perm
from itertools import (
    combinations, permutations, product, 
    combinations_with_replacement, chain, cycle
)

toppings = "🍕🧀🍄"  # 4 элемента: пицца, сыр, грибы, мясо
```

---

#### 1.1 Сочетания (combinations) — порядок не важен

**Математика:** \( C(n, k) = \frac{n!}{k!(n-k)!} \) — количество способов выбрать k элементов из n без учёта порядка.

```python
# Генерируем все сочетания по 2
pairs = combinations(toppings, 2)
print(*pairs)  
# ('🍕', '🧀') ('🍕', '🍄') ('🍕', '🍖') ('🧀', '🍄') ('🧀', '🍖') ('🍄', '🍖')

# Подсчитываем количество сочетаний
count = comb(4, 2)
print(count)  # 6
```

**Смысл:** вы заказываете пиццу с двумя разными начинками. Порядок, в котором их кладут, не имеет значения — 🍕+🧀 — это то же самое, что 🧀+🍕.

**Сленг:** «Сочетания — как дружеские объятия: кто первый, кто второй — не важно, результат один».

---

#### 1.2 Перестановки (permutations) — порядок важен

**Математика:** \( P(n, k) = \frac{n!}{(n-k)!} \) — количество способов выбрать и *упорядочить* k элементов из n.

```python
# Генерируем все перестановки по 2
ordered_pairs = permutations(toppings, 2)
print(*ordered_pairs)
# ('🍕', '🧀') ('🍕', '🍄') ('🍕', '🍖') 
# ('🧀', '🍕') ('🧀', '🍄') ('🧀', '🍖')
# ('🍄', '🍕') ('🍄', '🧀') ('🍄', '🍖')
# ('🍖', '🍕') ('🍖', '🧀') ('🍖', '🍄')

count_perm = perm(4, 2)
print(count_perm)  # 12
```

**Смысл:** вы выбираете две начинки *и* решаете, какая будет сверху, а какая снизу. 🍕+🧀 — это не то же самое, что 🧀+🍕.

**Сленг:** «Перестановки — как очередь в столовой: кто первый, тот и съел».

---

#### 1.3 Размещения с повторением (product) — декартово произведение

**Математика:** \( |A^n| = |A|^n \) — все возможные последовательности длины n из элементов множества (с повторениями).

```python
# Все пары с возможными повторениями (включая одинаковые начинки)
all_pairs = product(toppings, repeat=2)
print(*all_pairs)
# ('🍕', '🍕') ('🍕', '🧀') ('🍕', '🍄') ('🍕', '🍖')
# ('🧀', '🍕') ('🧀', '🧀') ('🧀', '🍄') ('🧀', '🍖')
# ('🍄', '🍕') ('🍄', '🧀') ('🍄', '🍄') ('🍄', '🍖')
# ('🍖', '🍕') ('🍖', '🧀') ('🍖', '🍄') ('🍖', '🍖')

count_product = 4 ** 2
print(count_product)  # 16
```

**Смысл:** вы можете выбрать *одну и ту же начинку дважды* (двойной сыр — да, двойные грибы — почему нет?). Плюс порядок важен.

**Сленг:** «Декартово произведение — это когда ты перебираешь все возможные пароли из 4-символьного алфавита. Да, включая 'aaaa'».

---

#### 1.4 Сочетания с повторениями (combinations_with_replacement)

**Математика:** \( C_R(n, k) = C(n + k - 1, k) = \frac{(n+k-1)!}{k!(n-1)!} \) — сочетания, где элементы могут повторяться, но порядок не важен.

```python
# Пары с возможными повторениями, но порядок не важен
pairs_with_replacement = combinations_with_replacement(toppings, 2)
print(*pairs_with_replacement)
# ('🍕', '🍕') ('🍕', '🧀') ('🍕', '🍄') ('🍕', '🍖')
# ('🧀', '🧀') ('🧀', '🍄') ('🧀', '🍖')
# ('🍄', '🍄') ('🍄', '🍖')
# ('🍖', '🍖')
```

**Смысл:** двойной сыр — разрешён, но 🧀+🍄 — это то же самое, что 🍄+🧀, поэтому они встречаются только один раз.

**Сленг:** «Сочетания с повторениями — как в мультиварке: порядок засыпки не важен, но можно положить два пакета одного супа».

### 2. Сравнительная таблица: все комбинаторные генераторы

| Функция                    | Смысл                                     | Повторы? | Порядок важен? | Количество для n=4, k=2 |
| --------------------------------- | ---------------------------------------------- | --------------- | -------------------------- | ------------------------------------ |
| `combinations`                  | Сочетания                             | ❌              | ❌                         | 6                                    |
| `permutations`                  | Перестановки                       | ❌              | ✅                         | 12                                   |
| `product`                       | Декартово произведение    | ✅              | ✅                         | 16                                   |
| `combinations_with_replacement` | Сочетания с повторениями | ✅              | ❌                         | 10                                   |

**Проверка на простом множестве `'ab'`:**

```python
from itertools import product, permutations, combinations

base = 'ab'

print("product:           ", *product(base, repeat=2))        # ('a','a') ('a','b') ('b','a') ('b','b')
print("permutations:      ", *permutations(base, 2))          # ('a','b') ('b','a')
print("combinations:      ", *combinations(base, 2))          # ('a','b')
print("combinations_wr:   ", *combinations_with_replacement(base, 2))  # ('a','a') ('a','b') ('b','b')
```

**Запомните (для собеседований):**

- `product` — самый мощный (все возможные варианты)
- `combinations` — самый ограниченный (без повторов, порядок не важен)
- `permutations` — без повторов, но порядок важен
- `combinations_with_replacement` — повторы разрешены, порядок не важен

### 3. Формулы подсчёта (из модуля math)

```python
from math import comb, perm

print(comb(4, 2))   # 6  — сочетания
print(perm(4, 2))   # 12 — перестановки (размещения)
print(4 ** 2)       # 16 — размещения с повторениями
```

**Формулы, которые должен знать каждый разработчик:**

| Конфигурация                         | Формула                             | math-функция         |
| ------------------------------------------------ | ------------------------------------------ | --------------------------- |
| Сочетания                               | \( C(n,k) = \frac{n!}{k!(n-k)!} \)         | `comb(n, k)`              |
| Перестановки                         | \( P(n,k) = \frac{n!}{(n-k)!} \)           | `perm(n, k)`              |
| Сочетания с повторениями   | \( C_R(n,k) = \frac{(n+k-1)!}{k!(n-1)!} \) | нет встроенной |
| Размещения с повторениями | \( n^k \)                                  | `n ** k`                  |

**Совет из опыта:** `comb(n, k)` и `perm(n, k)` доступны с Python 3.8+. Раньше приходилось писать свои реализации через `math.factorial`.

### 4. Бесконечные итераторы (осторожно, могут не остановиться)

#### 4.1 cycle() — бесконечный цикл

```python
from itertools import cycle

colors = ['red', 'green', 'blue']
color_cycle = cycle(colors)

# Будет крутиться вечно...
for _ in range(10):
    print(next(color_cycle))
# red, green, blue, red, green, blue, red, green, blue, red
```

**Реальный use-case:**

- Ротация цветов в графиках (палитра циклически повторяется)
- Раунд-робин schedule (распределение запросов между серверами)
- Генерация последовательностей для тестов

```python
# Пример: балансировщик нагрузки
servers = ['server1', 'server2', 'server3']
round_robin = cycle(servers)

for request_id in range(100):
    selected = next(round_robin)
    print(f"Request {request_id} → {selected}")
```

**⚠️ Важное предупреждение:** `cycle()` никогда не исчерпывается. Если использовать его в `list(cycle(...))` без ограничения — память закончится, а программа зависнет.

**Сленг:** «Cycle — как песня на репите, которая не кончается никогда. Останавливать придётся принудительно».

---

#### 4.2 count() — бесконечный арифметический счётчик

```python
from itertools import count

counter = count(start=10, step=2)

for _ in range(5):
    print(next(counter))
# 10, 12, 14, 16, 18
```

**Use-case:** индексация без ручного управления переменной.

---

#### 4.3 repeat() — повторение элемента

```python
from itertools import repeat

for item in repeat('ping', 3):
    print(item)
# ping, ping, ping
```

**Use-case:** создание итератора с одинаковыми значениями для `map()` с функцией.

### 5. chain() — склейка итераторов

```python
from itertools import chain

colors = ['red', 'green', 'blue']
sizes = ['small', 'medium', 'large']

# Склеиваем два списка в один поток
combined = chain(colors, sizes)

for item in combined:
    print(item)
# red, green, blue, small, medium, large
```

**Важное отличие от `+` для списков:**

```python
# ❌ list1 + list2 создаёт новый список (копирует память)
big_list = colors + sizes  # занимает память под оба списка

# ✅ chain создаёт итератор (ленивый, экономит память)
big_chain = chain(colors, sizes)  # почти бесплатно
```

**Академическая ремарка:**
`chain()` — это реализация ленивого конкатенирования (lazy concatenation). Экономит память для очень больших последовательностей.

**Сленг:** «Chain — как сцепка вагонов: поезд один, а вагоны из разных составов. И дополнительного места не нужно».

#### 6.2 Генерация всех подмножеств (power set)

```python
def powerset(iterable):
    """Возвращает все подмножества (power set) входного множества"""
    s = list(iterable)
    return chain.from_iterable(combinations(s, r) for r in range(len(s) + 1))

items = ['a', 'b', 'c']
for subset in powerset(items):
    print(subset)
# () ('a',) ('b',) ('c',) ('a','b') ('a','c') ('b','c') ('a','b','c')
```

**Use-case:** проверка всех возможных комбинаций параметров.

---

#### 6.3 Декартово произведение переменного количества итераторов

```python
# Ситуация: динамический список параметров
param_values = [
    [1, 2, 3],        # значение параметра A
    ['x', 'y'],       # B
    [True, False]     # C
]

# Генерируем все комбинации
for combo in product(*param_values):
    print(combo)
# (1, 'x', True), (1, 'x', False), (1, 'y', True), ...
# (3, 'y', False)
```

**Это классический кросс-валидационный перебор параметров (grid search) в data science.**

### 7. Реальный кейс: тестирование всех вариантов конфигурации

```python
def test_configuration(os, browser, resolution):
    print(f"Testing: {os} + {browser} + {resolution}")
    # реальная логика тестирования...

os_list = ['Windows', 'macOS', 'Linux']
browsers = ['Chrome', 'Firefox', 'Safari']
resolutions = ['1920x1080', '1280x720', '375x667']

# Полный кросс-прогон
for os, browser, resolution in product(os_list, browsers, resolutions):
    test_configuration(os, browser, resolution)

# Итого: 3 × 3 × 3 = 27 комбинаций
```

**Сленг:** «Проджект протестирован так, что ни один баг не спрячется. Потому что мы перебрали все варианты — как универсальный солдат, но без лишних движений».

### 9. Производительность и memory (важное замечание)

```python
# ❌ Плохо: материализуем всю комбинаторику в память
all_combos = list(combinations(range(100), 5))  # 75 млн элементов → memory error

# ✅ Хорошо: используем итератор лениво
for combo in combinations(range(100), 5):
    process(combo)  # память не взрывается
```

**Золотое правило:** `itertools` возвращает итераторы, а не списки. Не превращайте их в списки без острой необходимости. Обрабатывайте их прямо в цикле или передавайте в функции, работающие с итераторами.
