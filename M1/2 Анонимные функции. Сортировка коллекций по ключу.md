### 1. λ-исчисление в Python: анонимные функции (lambda)

**Академическое определение:**  
*Анонимная функция* (лямбда-функция) — это функция, определённая без идентификатора (имени), с собственной областью видимости, поддерживающая единственное выражение как тело. В Python она реализована через ключевое слово `lambda` (от λ-исчисления Чёрча).

**Сленговый перевод:**  
«Одноразовая функция на лету, без нейминга, только чистая жесть внутри одной строки».

---

#### 1.1 Классический pattern: map + lambda

Начнём с разогрева — вспомним `map`, но боевым.

```python
s = 'Hello Python from Russia'

# Обычный split
words = s.split()
print(words)  
# ['Hello', 'Python', 'from', 'Russia']

# Длины слов — через map + len
lengths = list(map(len, words))
print(lengths)  
# [5, 6, 4, 6]
```

А теперь — выражаем `len` через обёртку.

```python
# Через обычную функцию
def my_len2(x): 
    return len(x)

res2 = list(map(my_len2, words))

# Через лямбду, присвоенную переменной (полуанонимная)
my_len1 = lambda string: len(string)
res1 = list(map(my_len1, words))

# Чистая анонимная функция — прямо в map
res0 = list(map(lambda x: len(x), words))
```

**Резюме:** все три варианта работают одинаково. Но лямбда не загрязняет namespace одноразовыми именами.

---

#### 1.2 lambda как инструмент преобразования

```python
s = [1, 2, 3, 4, 5]

# map + lambda
res = list(map(lambda x: x ** 2, s))
print(res)  # [1, 4, 9, 16, 25]

# Тот же результат через генератор списков (list comp)
res2 = [x ** 2 for x in s]
```

> **Комментарий с опытом:** map + lambda — это функциональный стиль. Генератор списков — более питоничный. Но map быстрее для огромных данных и лучше читается в цепочках преобразований (pure pipelines).

---

#### 1.3 lambda через переменную (для понимания)

```python
lmd = lambda x: x ** 2   # теперь lmd — вызываемый объект
print(list(map(lmd, [1, 2, 3])))  # [1, 4, 9]
```

**Важно:** переменная хранит ссылку на функцию, но сама функция всё ещё анонимна по происхождению.

---

#### 1.4 lambda для нескольких аргументов

```python
add = lambda a, b: a + b
print(add(5, 3))  # 8
```

Или прямо в вызове:

```python
print((lambda a, b: a * b)(4, 5))  # 20
```

---

#### 1.5 Куда можно встроить lambda (и почему это страшно, но удобно)

Главное отличие lambda от `def`: **лямбда — это выражение**, а не инструкция.  
Её можно использовать *внутри* любых синтаксических конструкций.

```python
# Список функций (да, такое бывает)
funcs = [4, 5, lambda: print("аноним внутри списка"), 7, 8]
funcs[2]()   # вызов анонимной функции
```

**Ограничения (критично❗):**
- В теле лямбды только **одно выражение** (не блок инструкций).
- Нельзя использовать `return`, `assert`, `raise`, `:=` (walrus — иногда да, но осторожно).
- Нельзя писать многострочное тело.

```python
# SyntaxError:
lambda a: 
    print(a)   # так нельзя
```

**Сленг:** «лямбда — снайперский выстрел: один прицел, один выстрел, нет места для ветвлений».

### 2. lambda + key: умная сортировка

#### 2.1 Параметр key — фундаментальная концепция

Методы `list.sort()` и встроенная функция `sorted()` принимают параметр `key` — вызываемый объект (callable), который отображает каждый элемент на **значение для сравнения**.

**По умолчанию** сортируем по самим элементам:

```python
a = [4, 3, -10, 1, -7, 12]
print(sorted(a))  # [-10, -7, 1, 3, 4, 12]
```

**Кастомный key** — меняем логику сортировки.

#### 2.2 Пример: чётные сначала

Хотим: все чётные (0), затем нечётные (1).  
Key = чётность.

```python
def is_odd(x):
    return x % 2   # 0 для чётного, 1 для нечётного

b = sorted(a, key=is_odd)
print(b)  
# чётные идут первыми, порядок внутри групп сохраняется (stable sort)
```

**С лямбдой — лаконичнее:**

```python
b = sorted(a, key=lambda x: x % 2)
```

---

#### 2.3 Метод sort vs sorted

```python
a.sort(key=lambda x: x % 2)   # in-place
```

Поведение `key` идентично.

---

#### 2.4 Сортировка строк по разным признакам

```python
cities = ["Москва", "Тверь", "Смоленск", "Псков", "Рязань"]

# По длине слова
print(sorted(cities, key=len))
# ['Тверь', 'Псков', 'Москва', 'Рязань', 'Смоленск']

# По последней букве
print(sorted(cities, key=lambda x: x[-1]))
# ['Москва', 'Псков', 'Смоленск', 'Тверь', 'Рязань']

# По первой букве (лексикографически)
print(sorted(cities, key=lambda x: x[0]))
# ['Москва', 'Псков', 'Рязань', 'Смоленск', 'Тверь']
```

> **Заметка:** key применяется ровно один раз к каждому элементу (результат кэшируется). Это **декоративно-сортировочный паттерн** (Schwartzian transform).

---

#### 2.5 Сортировка сложных структур

📚 Кортеж с книгами: название, автор, цена.

```python
books = (
    ("Евгений Онегин", "Пушкин А.С.", 200),
    ("Муму", "Тургенев И.С.", 250),
    ("Мастер и Маргарита", "Булгаков М.А.", 500),
    ("Мертвые души", "Гоголь Н.В.", 190)
)

# Сортировка по цене (третий элемент)
sorted_books = sorted(books, key=lambda x: x[2])

for book in sorted_books:
    print(book)
```

Вывод:

```
('Мертвые души', 'Гоголь Н.В.', 190)
('Евгений Онегин', 'Пушкин А.С.', 200)
('Муму', 'Тургенев И.С.', 250)
('Мастер и Маргарита', 'Булгаков М.А.', 500)
```

**Академически:** key — это функция проекции.  
**Сленгово:** «Я сказал сортировщику — сравнивай не объекты целиком, а только их проекцию на нужную ось».

---

#### 2.6 Продвинутый пример: удельная стоимость + стабильность

Условие:  
Мешки с весом и стоимостью. Сортировка по **цене за кг** (удельной стоимости). При равной удельной стоимости — сначала мешки с меньшим весом.

```python
S = [
    [47, 470],
    [50, 600],
    [60, 480],
    [45, 540],
    [30, 300]
]

# Решение в лоб — через вспомогательный кортеж
S0 = [(x[1] / x[0], x[0], x[1]) for x in S]
S0.sort(key=lambda x: (x[0], x[1]))   # сначала удельная стоимость, потом вес

print(S0)
```

**Без вспомогательного списка** — в key вычисляем на лету:

```python
S.sort(key=lambda x: (x[1] / x[0], x[0]))
```

Но осторожно: вычисление будет работать много раз при сравнениях?  
Нет. `key` вычисляется один раз на элемент — **кэшируется** (спасибо timsort). Можно смело писать любые расчёты.

> **Совет с 20-летним стажем:** держи key простым и быстрым, если сортируются миллионы записей. Но для пары сотен элементов — даже логарифмы считай.

### Итоговая таблица (шпаргалка для бывалых и новичков)

| Конструкция                          | Тип использования                     | Сленг                                    |
|--------------------------------------|----------------------------------------|------------------------------------------|
| `lambda x: x**2`                     | функция-однострочник без имени        | «на лету наваял»                         |
| `sorted(data, key=func)`             | сортировка по ключу (однопроходная)    | «развернул проекцию»                     |
| `data.sort(key=lambda x: x[1])`      | in-place сортировка по полю           | «мутируем с мясом, но по уму»           |
| `key=lambda x: (x[0], -x[1])`        | составной ключ + обратный порядок     | «сначала А, потом убывающая Б»          |
